Neighbourhood-Inflated Seed Expansion Algorithm (NISE)

Variables

Name Standard value Explanation
seedCount 10 The number of seed which will be determined by the seed strategy.
probability 0.99 Is required in step 3 (Expansion) and must be between 0 and 1.
accuracy 0.00000001 Controls the size of the resulting communities: If it is large (e.g. $10^{-2}$) the resulting communities are small. If it is small (e.g. $10^{-8}$) the resulting communities are large.

Functionality

The Neighbourhood-Inflated Seed Expansion Algorithm (NISE) [WGDh16] is an overlapping community detection algorithm, which works on the method of seed expansion and therefore relies on finding good seeds efficiently. The four phases of the algorithm are: filtering, seeding, seed expansion and propagation.

  1. Filtering: Given a graph $G$, the filtering phase creates a subgraph $G'$, which has the property of remaining connected after removing any edge. The maximal subgraph with this property is called the "biconnected core".
  2. Seeding: The seeding strategy implemented here is called SpreadHubs. It greedily chooses an independent set of $k$ ($k$ is being set by the user in the variable "seedCount") nodes in the graph by looking at the degree of the nodes. In the beginning all nodes are unmarked and until $k$ nodes are found, the highest degree unmarked node is selected as a seed and its neighbours are being marked. If there is more than one node with a maximal degree, the algorithm takes an independent set of the unmarked nodes.
  3. Seed expansion: Now the seed communities need to be expanded. This is being done by a personalized Page Rank vector (also: random-walk with restart). With probability $p$ it takes a step of the random walk and with probability $1-p$ it jumps back to the a randomly chosen seed node. Therefore nodes, which are close to a seed node, are visited more often.
  4. Propagation: During the last phase, the nodes which have been detached in the filtering phase earlier are being reattached. Those whiskers are connected to $G$ by only one edge. Therefore they are being added to the communities which the node on the other side of that edge has.
